
# Project Overview
This project involves a detailed analysis of retail sales data using SQL. The primary goal is to explore, clean, and analyze retail sales patterns to uncover key business insights. The analysis covers customer behavior, sales trends across different time periods (shifts, months, and years), and product category performance.

# Dataset Description
The dataset includes transaction-level details of a retail store:

Sale_Date & Sale_Time: When the transaction occurred.

Customer_ID: Unique identifier for each customer.

Category: Product department (e.g., Clothing, Beauty, Electronics).

Total_Sale: The monetary value of the transaction.

Quantity & Price per Unit: Detailed breakdown of items sold.

# Tech Stack
Language: SQL (Structured Query Language)

Database: SQL Server / PostgreSQL

Key Techniques: CTEs (Common Table Expressions), Window Functions, Data Aggregation, and Case Statements.

# Key Business Questions Solved
The project addresses 10 critical business queries, including:

Basic Exploration: Identifying the total record count and unique customers.

Sales Trends: Calculating average sales per month and identifying the best-selling month for each year.

Customer Insights: Finding the top 5 customers based on total spending.

Category Analysis: Counting unique customers who purchased from each specific product category.

Shift Analysis: Using CASE statements to categorize sales into Morning, Afternoon, and Evening shifts to identify peak operational hours.

High-Value Transactions: Filtering orders with high sales volumes.

Time-Based Performance: Analyzing how sales vary across different years and months.
